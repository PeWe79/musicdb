Introduction

MusicDB is a music manager, server and player that focus on music. It is more like a music presenter than a database front-end.

Architecture

Features

MusicDB allows remote access to your music collection using a web-based user interface. You can listen to your music from anywhere via an audio stream. The stream is based on a song queue that can be managed by you or gets filled by a sophisticated random song selection algorithm.

The WebUI is focusing on being a presentation of your music rather than being a database front-end. When you are listening to your music, you do not work with software. Instead you explore your music collection.

Your Music

  • Visual Albums and songs are represented by their artwork. The UI color matches the album artwork.
  • Presentation MusicDB presents your music. It does not feel like a Database Front-end. No meta data overloaded list-based UI.
  • Fuzzy search The search allows you to have some typos when searching your music collection.
  • Focus on a Genre Hide all music not tagged with the genres you currently like to listen to.
  • Annotate your Mood Use custom flags to annotate songs with specific moods or themes.

Your Cloud

  • Private Your music is stored on your private server.
  • Everywhere Control via Web Application. Listen via audio stream.
  • Single User MusicDB is a Single-User Multi-Client application. Connect all your computers to MusicDB without messing around with user management.

Installation

MusicDB can be installed on any Linux Distribution. To use MusicDB properly you need to have the following dependencies installed:

  • Python 3.9+ for the back-end
  • Icecast and GStreamer for streaming
  • HTTP server like Apache for serving the Web User Interface

See the Install from Source documentation for a detailed list of all required packaged.
